# 设计规格：壁纸引擎

**日期**：2026-06-13
**文件**：`05-profile.html`（新增状态卡） + `08-wallpaper.html`（新增设置页）
**状态**：已批准（方案 B），进入实现

## 目标

在「我的」页增加「壁纸引擎」入口，让用户能把相册配置为定时轮换的壁纸源。**原型范围仅限配置 UI**——实际的 OS 级壁纸渲染是系统能力，不在演示范围。

引擎定位不是「选一张图当壁纸」（系统自带），而是「让相册自己动起来，每天/每次解锁看到不一样的回忆」。

## 选型结论

**B · 状态卡 + 独立设置页**

理由：
- 引擎是「常驻服务」型功能，需要持续可见的状态
- 我的页已有 storage-big / VIP 两张大卡，新增一张状态卡节奏自然
- 首次启用通过「未启用」状态的 CTA 引导，无需额外 wizard

淘汰方案：
- A · 单一入口行：状态不可见，用户不知道引擎在不在跑
- C · 首次引导流：实施复杂，原型 over-engineering

## 交互流程

### 首次启用
1. 我的页看到「未启用」卡
2. 点 → 推入 08-wallpaper.html
3. 默认选项已预填（当前选中相册「旅行 2025」+ 每次解锁 + 适配 + 高级全关）
4. 点底部「立即切换下一张」→ 引擎状态变 "启用中"
5. 返回我的页 → 卡片显示当前状态

### 日常调整
1. 我的页点状态卡 → 08-wallpaper
2. 调整任一选项 → 立即应用到顶部预览（不需保存按钮）
3. 点「立即切换」→ 预览 fade 200ms → 新图淡入
4. 可点相册更换 → 弹 sheet → 选 → 预览替换

### 暂停引擎
- 顶部 toggle 关闭 → Hero 卡显示「已暂停」灰底 0.5s
- 引擎状态保留，再次打开 toggle 恢复

## 状态与动画

| 触发 | 动效 |
|---|---|
| 启用/禁用 toggle | 预览淡出 150ms → 「已暂停/已停止」灰底 0.5s → 淡入回预览 |
| 切换下一张 | 旧预览 fade out 200ms → 新预览 fade in 200ms + 「下次切换」数字 tabular-nums 过渡 150ms |
| 切换方式变更 | 立即生效，队列圆点移动 200ms |
| 显示方式变更 | 预览图重新应用对应变换 200ms |
| 高级展开/收起 | 高度过渡 240ms ease-out |
| 弹 sheet（更换相册） | 从底滑入 280ms `cubic-bezier(0.32, 0.72, 0, 1)`，与 07-album 一致 |

## 设计系统对齐

- 复用 07-album 的底部 sheet（相册选择）
- 复用 storage-big / VIP 卡片节奏（圆角 16px / surface 底）
- 颜色严格走 `--accent` / `--fg` / `--muted` / `--surface` / `--accent-soft` / `--border`
- radio 沿用 06-files 的 iOS 蓝圈
- 数字统一 `font-variant-numeric: tabular-nums`
- 字体沿用 Inter，weight 仅用 400/500/600

## 不在范围（明确剔除）

- OS 级壁纸实际渲染（Android/iOS 系统能力）
- 电池 / 流量后台管理
- 多相册轮换队列（先做单相册，复杂逻辑下一版）
- 锁屏 vs 桌面分开配置
- 时间/天气感知切换（早上自动换风景照等）
- 真实照片数据接入（用 CSS 渐变模拟占位图）

## 实现约束

- 纯 vanilla JS（项目无框架）
- 数据硬编码在 `<script>` 内的常量数组
- Sheet 复用 07-album 的渲染逻辑：
  - **优先级 1**：抽取到公共模块（`__albumPicker`），两处共享
  - **优先级 2**（如果抽取成本过高）：在 08-wallpaper.html 内复制并标注 `// duplicated from 07-album, see refactor note`
- 不引外部资源
- 不修改 CSS token 体系
